# 内存泄漏修复报告

## 修复日期
2026-04-30

## 问题分析

### pprof heap 分析结果

从 `http://127.0.0.1:6060/debug/pprof/heap?debug=1` 获取的内存分析：

| 指标 | 值 | 说明 |
|------|-----|------|
| Alloc | 5.2 MB | 当前分配 |
| TotalAlloc | 22.8 MB | 总分配 |
| HeapObjects | 6,643 | 堆中存活对象 |
| Mallocs | 43,766 | 分配次数 |
| Frees | 37,123 | 释放次数 |
| GC次数 | 10 | 垃圾回收次数 |

### 主要内存热点

1. **io.Copy 数据转发** - ~425KB
   - `socks5.(*SOCKS5Client).ForwardData` - SOCKS5 数据转发缓冲区
   - `auth.Handle.func4` - 认证处理数据转发

2. **DNS 缓存 JSON 序列化** - 48 bytes
   - `dnscache.(*DNSCache).Save` - 定期保存缓存

3. **URL.String() 调用**
   - `utils.CheckShouldUseProxy` - 每次请求都创建新 URL 对象

### 根本原因

**连接资源未正确释放** - 在 WebSocket 和 SOCKS5 代理路径中，`server` 连接没有被 `defer server.Close()` 保护，导致连接泄漏。

## 修复内容

### auth/auth.go

| 修复位置 | 原代码 | 修复后 |
|---------|-------|--------|
| 第 291 行 | `server = clientConn` | 添加 `defer server.Close()` |
| 第 429 行 | `server = clientConn` | 添加 `defer server.Close()` |
| 第 439 行 | 已有 defer | 无变化 |

### simple/simple.go
