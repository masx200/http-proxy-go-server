## 1. 问题背景

在 DNS 解析模块持续运行过程中，进程内存与 goroutine 数量持续增长、不被回收，最终可能导致 OOM 或服务不可用。

**触发路径**：每次 DNS 查询（DoH / DoH3 协议）均会调用相关客户端函数，在高频查询场景下问题急剧放大。

**初始观测指标**（通过 pprof 采集）：

| 指标 | 数值 |
|------|------|
| 总 goroutine 数 | **1169** |
| `persistConn.readLoop` | **112** 个 |
| `persistConn.writeLoop` | **112** 个 |
| QUIC 相关 goroutine（估算） | **~640** 个 |

```
  github.com/quic-go/quic-go.(*connection).runSendQueue
```

## 3. 根因分析

### 3.1 DohClient —— http.Transport 泄漏

**文件**：`dns_experiment/dns.go`  
**函数**：`DohClient()`

#### 问题代码（修复前）

```go
// ❌ 旧代码：每次调用都创建全新的 http.Transport
func DohClient(msg *dns.Msg, dohServerURL string, dohip string,
    Proxy func(*http.Request) (*url.URL, error),
    tranportConfigurations ...func(*http.Transport) *http.Transport) (*dns.Msg, error) {

    if len(dohip) > 0 {
        transport := &http.Transport{...}      // 每次 new
        client = &http.Client{Transport: transport}
    } else {
        // ⚠️ 严重 Bug：直接修改全局 http.DefaultClient.Transport！
        http.DefaultClient.Transport = &http.Transport{...}
        client = http.DefaultClient
    }
    // 旧 transport 不会被关闭，持有的连接和 goroutine 永久泄漏
}
```

#### 泄漏机制

```
每次 DohClient 调用
    └── new(http.Transport)
            ├── 建立 TCP/TLS 连接
            ├── 启动 persistConn.readLoop  (goroutine，永不退出)
            └── 启动 persistConn.writeLoop (goroutine，永不退出)

旧 Transport 被 GC 前无法关闭连接池
→ 每次调用 +2 个 goroutine（readLoop + writeLoop）
→ 112 次调用 = 224 个泄漏 goroutine
```

#### 并发 Bug

`else` 分支直接写 `http.DefaultClient.Transport`：
- 多个 goroutine 并发调用时存在**数据竞争**（race condition）
- 新配置会覆盖前一次配置，导致行为不可预期

### 3.2 Doh3nslookup —— quic.Transport 泄漏

**文件**：`doh/doh3.go`  
**函数**：`Doh3nslookup()` → `doHTTP3Client()` → `h3_experiment.DoHTTP3Client()`

#### 问题代码（修复前）

```go
// ❌ 旧代码：每次查询都调用外部函数，内部每次创建新的 UDP socket + quic.Transport
func doHTTP3Client(msg *dns.Msg, dohttp3ServerURL string, dohip ...string) (*dns.Msg, error) {
    // h3_experiment.CreateHTTP3TransportWithIP 内部：
    //   net.ListenUDP("udp", nil)          ← 每次开新 UDP socket
    //   quic.Transport{Conn: udpConn}      ← 每次创建新 Transport
    //   从不调用 qt.Close() 或 h3tr.Close()
    client, _ := h3_experiment.DoHTTP3Client(...)
    // ...
}
```

#### 泄漏机制

```
每次 Doh3nslookup 调用
    └── 外部包 CreateHTTP3TransportWithIP
            ├── net.ListenUDP()            → 新 UDP socket（系统资源）
            └── quic.Transport{}
                    ├── Transport.listen   (goroutine，永不退出)
                    ├── sendQueue.Run      (goroutine，永不退出)
                    ├── connection.run     (goroutine，永不退出)
                    └── connection.runSendQueue (goroutine，永不退出)

每次 DNS 查询 +4 个 goroutine + 1 个 UDP socket
→ 160 次查询 = 640 个泄漏 goroutine + 160 个 fd 泄漏
```

## 4. 修复方案

### 核心思路

> **复用，不重建。** Transport / quic.Transport 是重量级资源，应全局缓存并按参数组合复用。

---

### 4.1 dns_experiment/dns.go 修复

#### 修复策略

1. 新增 `sync.Map` 全局缓存，以 `(serverURL | dohip | hasProxy)` 为 key
2. 将创建逻辑提取到 `getOrCreateDohClient()`，使用 `LoadOrStore` 保证并发安全
3. `else` 分支改为创建独立 Transport，彻底消除全局变量并发修改

#### 新增数据结构

```go
// 缓存已创建的 http.Client，按 (serverURL+dohip+hasProxy) 为 key 复用连接池
var dohClientCache sync.Map

type dohClientCacheEntry struct {
    client   *http.Client
    lastUsed time.Time
    mu       sync.Mutex
}

// getDohClientCacheKey 生成缓存键
func getDohClientCacheKey(dohServerURL, dohip string, hasProxy bool) string {
    proxyPart := "noproxy"
    if hasProxy {
        proxyPart = "proxy"
    }
    return dohServerURL + "|" + dohip + "|" + proxyPart
}
```

#### getOrCreateDohClient 核心逻辑

```go
func getOrCreateDohClient(dohServerURL, dohip string,
    Proxy func(*http.Request) (*url.URL, error),
    tranportConfigurations ...func(*http.Transport) *http.Transport) *http.Client {

    cacheKey := getDohClientCacheKey(dohServerURL, dohip, Proxy != nil)

    // 快路径：缓存命中直接返回
    if v, ok := dohClientCache.Load(cacheKey); ok {
        entry := v.(*dohClientCacheEntry)
        entry.mu.Lock()
        entry.lastUsed = time.Now()
        entry.mu.Unlock()
        return entry.client
    }

    // 慢路径：创建新 Client（含完整 Transport 配置）
    var client *http.Client
    if len(dohip) > 0 {
        transport := &http.Transport{
            ForceAttemptHTTP2:   true,
            MaxIdleConns:        10,
            MaxIdleConnsPerHost: 5,
            IdleConnTimeout:     90 * time.Second,
        }
        transport.DialTLSContext = buildDialTLSContext(dohip, dohServerURL, transport)
        transport.DialContext    = buildDialContext(dohip, dohServerURL, transport)
        if Proxy != nil { transport.Proxy = Proxy }
        for _, f := range tranportConfigurations { transport = f(transport) }
        client = &http.Client{Transport: transport, Timeout: 30 * time.Second}
    } else {
        // ✅ 修复：创建独立 Transport，不再修改 http.DefaultClient
        transport := &http.Transport{
            ForceAttemptHTTP2:   true,
            MaxIdleConns:        10,
            MaxIdleConnsPerHost: 5,
            IdleConnTimeout:     90 * time.Second,
        }
        if Proxy != nil { transport.Proxy = Proxy }
        for _, f := range tranportConfigurations { transport = f(transport) }
        client = &http.Client{Transport: transport, Timeout: 30 * time.Second}
    }

    entry := &dohClientCacheEntry{client: client, lastUsed: time.Now()}

    // LoadOrStore 保证并发安全：若已有其他 goroutine 先存入，使用已有的
    if actual, loaded := dohClientCache.LoadOrStore(cacheKey, entry); loaded {
        return actual.(*dohClientCacheEntry).client
    }
    return client
}
```

#### DohClient 简化后

```go
func DohClient(msg *dns.Msg, dohServerURL, dohip string,
    Proxy func(*http.Request) (*url.URL, error),
    tranportConfigurations ...func(*http.Transport) *http.Transport) (*dns.Msg, error) {

    msg.Id = 0
    // ... 构建 req ...

    // ✅ 复用缓存 client，不再每次 new Transport
    client := getOrCreateDohClient(dohServerURL, dohip, Proxy, tranportConfigurations...)

    res, err := client.Do(req)
    // ... 处理响应 ...
}
```

### 4.2 doh/doh3.go 修复

#### 修复策略

1. **完全重写**，不再依赖外部泄漏包 `h3_experiment.DoHTTP3Client`
2. 自行管理 `quic.Transport` 生命周期，全局缓存复用
3. `getOrCreateH3Client` + `LoadOrStore` 并发安全创建

#### 新增数据结构

```go
// h3ClientEntry 缓存的 H3 客户端条目，持有长生命周期的 quic.Transport
type h3ClientEntry struct {
    client        *http.Client
    quicTransport *quic.Transport
    h3Transport   *http3.Transport
    mu            sync.Mutex
}

// h3ClientCache 按 (dohurl|dohip) 为 key 缓存 H3 客户端
var h3ClientCache sync.Map
```

#### getOrCreateH3Client 核心逻辑

```go
func getOrCreateH3Client(dohurl, dohip string) *h3ClientEntry {
    cacheKey := dohurl + "|" + dohip

    // 快路径
    if v, ok := h3ClientCache.Load(cacheKey); ok {
        return v.(*h3ClientEntry)
    }

    // 慢路径：创建新 quic.Transport
    entry := createH3Client(dohurl, dohip)

    // LoadOrStore 防止并发重复创建
    if actual, loaded := h3ClientCache.LoadOrStore(cacheKey, entry); loaded {
        // ✅ 关闭多余创建的 Transport，释放 UDP socket
        entry.quicTransport.Close()
        entry.h3Transport.Close()
        return actual.(*h3ClientEntry)
    }
    return entry
}

func createH3Client(dohurl, dohip string) *h3ClientEntry {
    udpConn, err := net.ListenUDP("udp", nil)
    if err != nil {
        // 降级：不含 quic.Transport 的基础 entry
        h3tr := &http3.Transport{}
        return &h3ClientEntry{
            client:      &http.Client{Transport: h3tr, Timeout: 30 * time.Second},
            h3Transport: h3tr,
        }
    }

    qt := &quic.Transport{Conn: udpConn}

    // 按需配置自定义 IP 拨号
    var dialFunc func(ctx context.Context, addr string,
        tlsConf *tls.Config, quicConf *quic.Config) (*quic.Conn, error)
    if dohip != "" {
        ip := dohip
        dialFunc = func(ctx context.Context, addr string,
            tlsConf *tls.Config, quicConf *quic.Config) (*quic.Conn, error) {
            _, port, _ := net.SplitHostPort(addr)
            a, _ := net.ResolveUDPAddr("udp", net.JoinHostPort(ip, port))
            return qt.DialEarly(ctx, a, tlsConf, quicConf) // ✅ 复用 qt
        }
    } else {
        dialFunc = func(ctx context.Context, addr string,
            tlsConf *tls.Config, quicConf *quic.Config) (*quic.Conn, error) {
            a, _ := net.ResolveUDPAddr("udp", addr)
            return qt.DialEarly(ctx, a, tlsConf, quicConf) // ✅ 复用 qt
        }
    }

    h3tr := &http3.Transport{Dial: dialFunc}
    return &h3ClientEntry{
        client:        &http.Client{Transport: h3tr, Timeout: 30 * time.Second},
        quicTransport: qt,
        h3Transport:   h3tr,
    }
}
```

## 5. 修复效果对比

### goroutine 数量对比

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| 100 次 DoH 查询 | +200 goroutine（readLoop/writeLoop） | **+0**（连接池复用） |
| 100 次 DoH3 查询 | +400 goroutine（QUIC 相关） | **+0**（Transport 复用） |
| 1000 次混合查询 | ~3000+ goroutine | **稳定在初始值附近** |
| 并发安全 | ❌ DefaultClient 数据竞争 | ✅ LoadOrStore 无竞争 |

### 内存对比

| 资源 | 修复前（100次查询） | 修复后（100次查询） |
|------|---------------------|---------------------|
| http.Transport 实例 | 100 个 | **1 个**（按参数组合缓存） |
| quic.Transport 实例 | 100 个 | **1 个**（按参数组合缓存） |
| UDP socket（fd） | 100 个 | **1 个** |
| TLS 连接 | 不可复用 | ✅ 连接池复用 |

### 架构对比图

```
修复前（泄漏）：
DNS查询 → DohClient() → new(Transport) → goroutine×2 → 泄漏
DNS查询 → DohClient() → new(Transport) → goroutine×2 → 泄漏
DNS查询 → DohClient() → new(Transport) → goroutine×2 → 泄漏

修复后（复用）：
               ┌──── 全局缓存 (sync.Map) ────┐
DNS查询 ──────►│  Transport A (server1)       │
DNS查询 ──────►│  Transport B (server2)       │
DNS查询 ──────►│  Transport A (server1) 命中  │
               └─────────────────────────────┘
                只有首次创建 goroutine，后续全部复用
```

## 7. 最佳实践总结

### ✅ Go 中正确管理 HTTP 客户端

```go
// ✅ 正确：全局复用 http.Client / http.Transport
var client = &http.Client{
    Transport: &http.Transport{
        MaxIdleConns:        100,
        MaxIdleConnsPerHost: 10,
        IdleConnTimeout:     90 * time.Second,
    },
}

// ❌ 错误：每次调用创建新 client
func doRequest() {
    client := &http.Client{} // Transport 默认 nil，使用 DefaultTransport
    // ...                   // 但实际上 DefaultTransport 是全局的，此处安全
                             // 若显式 new(Transport) 则必然泄漏
}
```

### ✅ 并发安全的全局缓存

```go
var cache sync.Map

func getOrCreate(key string) *Resource {
    if v, ok := cache.Load(key); ok {
        return v.(*Resource)
    }
    r := newExpensiveResource()
    if actual, loaded := cache.LoadOrStore(key, r); loaded {
        r.Close() // 关闭竞争中多创建的资源
        return actual.(*Resource)
    }
    return r
}
```

### ✅ quic.Transport 生命周期管理

```go
// ✅ 正确：复用 quic.Transport（底层 UDP socket）
var qt = &quic.Transport{Conn: mustListenUDP()}

func dial(addr string) (*quic.Conn, error) {
    return qt.DialEarly(ctx, resolvedAddr, tlsCfg, quicCfg)
}

// ❌ 错误：每次创建新的 quic.Transport
func badDial(addr string) {
    udpConn, _ := net.ListenUDP("udp", nil)   // fd 泄漏
    qt := &quic.Transport{Conn: udpConn}       // goroutine 泄漏×4
    // 没有 defer qt.Close()
}
```

### ✅ 通用检查清单

| 场景 | 检查项 |
|------|--------|
| HTTP 客户端 | `http.Transport` 是否全局复用？ |
| QUIC 客户端 | `quic.Transport` 是否全局复用？UDP socket 是否复用？ |
| goroutine | 每个 goroutine 是否有明确的退出条件？ |
| 全局变量 | 是否并发写全局变量（如 `http.DefaultClient`）？ |
| 资源关闭 | 创建资源时是否有对应的 `Close()`？ |
